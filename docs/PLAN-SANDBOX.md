# Sandbox Terraform Parameterisation Plan

## Goal

I want:

* To learn terraform without accidentally causing problems with production
* To experiment in a sandbox (either a sub AWS account or my own account)
* To be selective about what resources are built to be cost-effective
* Only do the minimal necessary changes to the terraform configuration to support this
* This to be useful to the OAF development team

## Solution Overview

* Each developer or environment has a named **profile** — e.g. `PROFILE=oaf`, `PROFILE=ian`
* Profile configuration lives in `profiles/$(PROFILE)/` — three files with different
  sensitivity and lifecycle
* `PROFILE=` is required on all terraform make targets when more than one profile exists;
  if only one exists it is used automatically; if none exist, ansible works as before
  and terraform will complain about missing variables
* A Ruby executable inventory script replaces the static `inventory/ec2-hosts`, reading
  profile files and querying AWS/Linode APIs to build inventory dynamically
* `tf-destroy` is the only new make target; existing `tf-apply`/`tf-plan`/`tf-init`
  are unchanged in behaviour
* Documented in `docs/SANDBOX.md` — practical how-to only, references this file

## Generated Files

Make generates these from profile files. All gitignored.

### `terraform/$(PROFILE).tfvars`

From `profiles/$(PROFILE)/secrets` + `profiles/$(PROFILE)/config`.
Converts `key="value"` shell format to `key = "value"` tfvars format,
and `key=true/false` to `key = true/false`:

```makefile
terraform/$(PROFILE).tfvars: profiles/$(PROFILE)/secrets profiles/$(PROFILE)/config
	cat profiles/$(PROFILE)/secrets profiles/$(PROFILE)/config | \
	  sed 's/^\([A-Za-z_][^=]*\)="\(.*\)"$$/\1 = "\2"/' | \
	  sed 's/^\([A-Za-z_][^=]*\)=\(true\|false\)$$/\1 = \2/' | \
	  grep -v '^#' | grep -v '^$$' > $@
```

### `ansible/group_vars/terraform_$(PROFILE).yml`

Only whitelisted non-secret config vars plus AWS credentials (needed by site.yml
for RDS queries), prefixed `terraform_` so source is obvious in ansible:

```makefile
ansible/group_vars/terraform_$(PROFILE).yml: profiles/$(PROFILE)/config profiles/$(PROFILE)/secrets
	echo "# Generated from profiles/$(PROFILE)/ - do not edit" > $@
	sed -n 's/^\(base_domain\)="\(.*\)"$$/terraform_\1: "\2"/p' \
	  profiles/$(PROFILE)/config >> $@
	sed -n 's/^\(aws_access_key\|aws_secret_key\)="\(.*\)"$$/terraform_\1: "\2"/p' \
	  profiles/$(PROFILE)/secrets >> $@
```

## Makefile Changes

### PROFILE Detection

```makefile
PROFILE_DIRS  := $(wildcard profiles/*/config)
PROFILE_COUNT := $(words $(PROFILE_DIRS))

ifeq ($(PROFILE_COUNT),0)
  TF_VAR_FILE :=
else ifeq ($(PROFILE_COUNT),1)
  PROFILE     := $(patsubst profiles/%/config,%,$(PROFILE_DIRS))
  TF_VAR_FILE := -var-file=terraform/$(PROFILE).tfvars
else
ifeq ($(origin PROFILE),undefined)
  $(error PROFILE is required when multiple profiles exist, e.g. PROFILE=ian)
endif
  TF_VAR_FILE := -var-file=terraform/$(PROFILE).tfvars
endif

tf_state_bucket = $(shell grep ^terraform_state_bucket profiles/$(PROFILE)/config | cut -d'"' -f2)
tf_region       = $(shell grep ^ec2_region profiles/$(PROFILE)/config | cut -d'"' -f2)
```

### Updated terraform targets

```makefile
tf-init: terraform/$(PROFILE).tfvars
	terraform -chdir=terraform init \
	  -backend-config="bucket=$(tf_state_bucket)" \
	  -backend-config="region=$(tf_region)"

tf-plan: terraform/$(PROFILE).tfvars ansible/group_vars/terraform_$(PROFILE).yml
	terraform -chdir=terraform plan $(TF_VAR_FILE)

tf-apply: terraform/$(PROFILE).tfvars ansible/group_vars/terraform_$(PROFILE).yml
	terraform -chdir=terraform apply $(TF_VAR_FILE)
```

### New `tf-destroy` target

```makefile
tf-destroy:
ifeq ($(PROFILE),oaf)
	$(error tf-destroy refused for PROFILE=oaf -- too dangerous!)
endif
	@echo "This is the command to destroy what this profile created."
	@echo "Execute it manually if that is really what you want to do:"
	@echo ""
	@echo "  terraform -chdir=terraform destroy $(TF_VAR_FILE)"
	@echo ""
```

### New `update-hosts` target

```makefile
update-hosts: profiles/$(PROFILE)/secrets profiles/$(PROFILE)/config
	ruby inventory/update-hosts.rb $(PROFILE)
```

## Ruby Inventory Script

Replaces `inventory/ec2-hosts`. Executable Ruby script, outputs ansible INI format.
Read by ansible via `ansible.cfg` `inventory = inventory` setting.

### AWS Name tag to ansible group mapping

```ruby
NAME_TAG_MAP = {
  "openaustralia"          => { group: "openaustralia_old" },
  "openaustralia-prod"     => { group: "openaustralia_new" },
  "au.proxy"               => { group: "proxy" },
  "theyvoteforyou"         => { group: "theyvoteforyou" },
  "metabase"               => { group: "metabase" },
  "righttoknow-staging"    => { group: "righttoknow_staging" },
  "righttoknow-production" => { group: "righttoknow_production" },
  "openvpn-server"         => { group: "openvpn", host: "openvpn1",
                                vars: { ansible_user: "ubuntu",
                                        ansible_ssh_private_key_file: "terraform.pem" } },
}
```

### Discovery logic

1. Read `PROFILE` from environment, load `profiles/$(PROFILE)/config` if present
2. Fall back to `base_domain=org.au` if no profile
3. For hosts in NAME_TAG_MAP: emit `<hostname>.<base_domain>` as ansible host
4. For openvpn: query AWS API for instance with Name tag `openvpn-server`,
   use public IP as `ansible_ssh_host`, emit logical name `openvpn1`
5. For planningalerts: query AWS for instances tagged `Application=planningalerts`
   AND `BlueGreen=$(planningalerts_colour)`, use their EC2 DNS names;
   fall back to `profiles/$(PROFILE)/hosts` if AWS credentials unavailable
6. For Linode (morph, cuttlefish): query Linode API by label if token available
7. Emit full group hierarchy matching current `ec2-hosts` structure including
   `children` groups (`openaustralia`, `righttoknow`, `requires_mysql` etc.)

### `inventory/update-hosts.rb`

Separate script called by `make update-hosts PROFILE=ian`. Queries AWS for
planningalerts instances using credentials from `profiles/$(PROFILE)/secrets`.
Updates `profiles/$(PROFILE)/hosts` without clobbering values it cannot discover.
Safe to run repeatedly.

## Vagrantfile Refactor

Vagrantfile requires the inventory script as a Ruby library for NAME_TAG_MAP.
Uses `base_domain="test"` hardcoded (RFC 2606 reserved TLD, no profile needed).

Validates every host in NAME_TAG_MAP is either assigned a static IP or
explicitly listed in IGNORED_HOSTS. Raises loudly if a new service is added
to terraform (and therefore NAME_TAG_MAP) without updating the Vagrantfile:

```ruby
STATIC_IPS = {
  "openaustralia"        => "192.168.56.22",
  "openaustralia-prod"   => "192.168.56.23",
  "au.proxy"             => "192.168.56.12",
  "theyvoteforyou"       => "192.168.56.26",
  "metabase"             => "192.168.56.20",
  "righttoknow-staging"  => "192.168.56.25",
}

IGNORED_HOSTS = %w[
  righttoknow-production   # too resource heavy for vagrant
  openvpn-server           # not needed locally
  morph                    # linode, not vagrant
  cuttlefish               # linode, not vagrant
]

NAME_TAG_MAP.each_key do |name|
  unless STATIC_IPS.key?(name) || IGNORED_HOSTS.include?(name)
    raise "#{name} has no static IP and is not in IGNORED_HOSTS -- update Vagrantfile!"
  end
end
```

Ubuntu version per host and vagrant-specific aliases remain explicit in Vagrantfile.

## Ansible Variable Override Pattern

Vaulted variables renamed with `default_` prefix. A non-vaulted variable
picks up the `terraform_*` override from the generated group_vars file if
present, otherwise falls back to the vault value. No role changes required.

```yaml
# group_vars/all.yml
default_rds_admin_password: !vault |
    $ANSIBLE_VAULT;1.2;AES256;all
    ..redacted..

rds_admin_password: "{{ terraform_rds_admin_password | default(default_rds_admin_password) }}"
```

```yaml
# group_vars/ec2.yml
default_aws_access_key: !vault |
    $ANSIBLE_VAULT;1.2;AES256;ec2
    ..redacted..

aws_access_key: "{{ terraform_aws_access_key | default(default_aws_access_key) }}"
aws_secret_key: "{{ terraform_aws_secret_key | default(default_aws_secret_key) }}"
```

```yaml
# ansible/group_vars/terraform_ian.yml (generated, gitignored)
# Generated from profiles/ian/ - do not edit
terraform_base_domain: "thesite.info"
terraform_aws_access_key: "AKIAIOSFODNN7EXAMPLE"
terraform_aws_secret_key: "wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY"
```

### Vault access in sandbox

Ansible only decrypts vault values for groups relevant to the targeted hosts.
Existing vault identity mechanism handles partial access via `.envrc`:

```bash
# ansible.cfg (production default)
vault_identity_list = .vault_pass.txt,rtk@.rtk-vault-pass,ec2@.ec2-vault-pass,all@.all-vault-pass

# Developer .envrc (omit identities they don't have)
export ANSIBLE_VAULT_IDENTITY_LIST=".vault_pass.txt,ec2@.ec2-vault-pass,all@.all-vault-pass"
```

Sandbox workflow for missing variables:
```
run make check-<service> PROFILE=ian
  -> if it complains about a missing/undecryptable variable
  -> add that variable to profiles/ian/secrets
  -> re-run
```

### Token-gating

Automatically disables services when provider token is absent, with warning:

```hcl
locals {
  linode_available  = var.linode_api_token != null && var.linode_api_token != ""
  enable_morph      = local.linode_available && var.enable_morph
  enable_cuttlefish = local.linode_available && var.enable_cuttlefish
}

check "morph_token_missing" {
  assert {
    condition     = !var.enable_morph || local.linode_available
    error_message = "WARNING: enable_morph=true but linode_api_token not set -- morph skipped"
  }
}
```

Linode provider requires non-null token even when all linode modules are disabled.
Use `linode_api_token="dummy"` in `profiles/ian/secrets` when not needed.

### Module count pattern

```hcl
module "openaustralia" {
  count  = var.enable_openaustralia ? 1 : 0
  source = "./openaustralia"
  ...
}
```

Module outputs become `module.openaustralia[0].main_public_ip` etc.
Modules are grouped by server/purpose so each module owns its own DNS records,
minimising cross-module reference issues.

### Domain parameterisation

Replace hardcoded `org.au` with `var.base_domain` in all dns.tf files.
Pass `base_domain = var.base_domain` into each module call in `main.tf`.
Affected modules: `openaustralia`, `righttoknow`, `planningalerts`,
`theyvoteforyou`, `morph`, `oaf`, `raisely`, `campaign-monitor`, `social`, `proxy`.

### S3 bucket namespacing

```hcl
resource "aws_s3_bucket" "oaf-backups" {
  bucket = "${var.s3_bucket_prefix}-backups"
}
```

Affects: `oaf-backups`, `oaf-elasticsearch-snapshots`, `oaf-terraform-state`,
`planningalerts-as-production`.

### IAM namespacing

Using separate AWS accounts per profile means IAM names do not clash.
No changes needed unless sharing an account.
